# Fit & Flare Studio
> **A Unified Digital Tailoring Ecosystem**

## üö´ Problem Statement
The apparel and tailoring ecosystem involving customers, designers, tailors, and fabric merchants operates in a highly fragmented manner. Each stakeholder functions independently, relying on informal communication, manual coordination, and disconnected workflows.

**Key Pain Points:**
*   **Design Misinterpretation**: Customers struggle to translate ideas; tailors face ambiguity.
*   **Inefficient Collaboration**: disconnected workflows between designers and tailors.
*   **Material Mismatches**: Fabric merchants lack visibility into actual demand, causing waste.
*   **Operational Inefficiency**: Repeated alterations, delays, and lack of tracking.
*   **No Data Traceability**: Absence of scalable growth models or analytics.

There is currently no unified digital platform that connects all stakeholders into a single, transparent, and structured workflow.

---

## üéØ Project Goal
To build a web-based digital tailoring ecosystem that seamlessly connects customers, designers, tailors, and fabric merchants on a single platform.

**Objectives:**
*   Standardize communication across all stakeholders.
*   Enable accurate design execution and measurement handling.
*   Improve material planning and availability.
*   Increase operational efficiency and transparency.
*   Create a scalable, data-driven fashion service ecosystem.

---

## üí° Proposed Solution
Fit & Flare Studio is designed as an end-to-end digital orchestration platform that integrates design creation, material sourcing, tailoring execution, and customer engagement into one cohesive system.

### Core Interface Components:
1.  **Customer Interface**:
    *   Visual blouse customization wizard.
    *   Measurement capture with validation.
    *   Order placement, tracking, and alteration requests.
2.  **Designer Interface**:
    *   Upload standardized blouse design templates.
    *   Define constraints and fabric recommendations.
3.  **Tailor Interface**:
    *   Access finalized designs and measurements.
    *   Manage order queues and update stitching progress.
4.  **Merchant Interface**:
    *   Fabric catalog management.
    *   Demand signal visibility for inventory planning.
5.  **Admin Platform**:
    *   Rule-based pricing, lifecycle management, and analytics.

## üõ†Ô∏è Technical Architecture & Setup

### Technology Stack
*   **Frontend**: React (Vite) + Vanilla CSS (Premium Design System)
*   **Backend**: Node.js + Express.js
*   **Database**: PostgreSQL (Structured Relational Data)

### Getting Started

#### Prerequisites
*   Node.js (v16+)
*   PostgreSQL (v14+)

#### Installation
1.  **Database**:
    *   Run `setup_db_manual.bat` to initialize schema and seed data.
2.  **Backend**:
    ```bash
    cd backend
    npm install
    npm run dev
    ```
3.  **Frontend**:
    ```bash
    cd frontend
    npm install
    npm run dev
    ```

For detailed system flow and architecture, refer to `SYSTEM_FLOW.md` and `ARCHITECTURE.md`.
